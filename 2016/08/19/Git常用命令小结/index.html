Git常用命令小结

创建本地Git库(以https协议为例)：git clone <版本库网址> <本地目录名>

git clone https://8.8.8.8/start/sandbox.git

Git clone –help [获取详细帮助信息]

本地clone是远端git库的镜像拷贝，本地目录结构如下
.git/

——/objects:存放的是实际文件

——/index: 一个索引文件，存放的暂存区的整个目录树信息，并且为目录树中的每个文件都保存了时间戳和长度

——/HEAD: 文件保存了当前的branch

——/refs:称为引用目录，引用就是一个文件，里面包好了一个commit id

——/…

本地文件的修改与提交

git add . /添加新增or改动文件/

git commit /提交文件更新到本地库/

git commit -m /添加提交注释/
git commit -a /跳过暂存区, 将未add的文件一并提交/
git commit –amend /更新最后一次提交的commit，如修改注释等*/

git log ／查看当前分支的提交历史/

合并一个分支修改到另一个分支上:

在本地 master 分支上做了一个commit ( 38361a68138140827b31b72f8bbfd88b3705d77a ) ， 如何把它放到 本地develop分支上？

git checkout develop
git cherry-pick 38361a68

拉代码时出现：“git merge ‘ XXXXXX”等一个新的commit信息：

git pull –rebase
git pull –rebase 防止了菱形提交的形成, 保持提交曲线为直线方便大家理解。
git pull -> git fetch + git merge
git pull –rabase -> git fetch + git rebase
http://blog.sina.com.cn/s/blog_88243b5e0101j38v.html

打标签：

git tag -a v1.2 -m”my version 1.4”
保全全部的拷贝。资源、邮件、数据等。
git tag v1.4-lw
just a tag no information .
查看标签：
git tag

查看当前全路径url:

git remote

在线上的一个版本有个bug要修改

在本地新建一个分支：
git checkout -b hotfix
// do something you will do
git commit -a -m”fix the email address”
切回项目分支如：
git checkout master
git merge hotfix
最后，删除这一临时分支
git branch -d hotfix

[git checkout -b hotfix] equal
git branch hotfix
git checkout hotfix

如果在merge中出现冲突，如何解决？

git status
解决冲突
git status 看提示语句。一切ok 则
git commit -m”fix conflict “
一切提示OK, push or do something you want

查看本地未使用，或者未merger的分支

git branch —no-merged
不需要则删除它
git branch -d xxxxx

如果工作目录的本地代码做了修改但尚未提交，pull 拉取远程仓库的新提交时，往往会提示冲突：

git pull
error: Your local changes to the following files would be overwritten by merge:
/there/is/a/conflict/file
Please, commit your changes or stash them before you can merge.
如上所示，有 commit 和 stash 两种处理方法。针对本地代码的完成情况我们需要作出选择。

代码已完成

如果确认本地代码已经完成无误，可以先将本地代码 commit 到本地仓库。再次 pull 拉取远程仓库时，如无冲突，Git 会自动产生一次“合并”提交：

代码未完成

但如果本地代码仍未完成，此时推荐使用 stash 命令暂存修改，避免将未完成的功能代码 commit 到本地仓库，污染仓库。
暂存当前修改
第一步，stash 暂存当前修改：
git stash save “填写你的备注”
Saved working directory and index state ……
git status
On branch master
nothing to commit, working directory clean
可以看到暂存后工作目录一干二净。这是因为 stash 命令可以将“修改过的被追踪的文件（modified tracked files）”和“暂存的变更（staged changes）”暂存到临时堆栈中，并将工作目录还原干净，以便后续的操作。
Stashing takes the dirty state of your working directory – that is, your modified tracked files and staged changes – and saves it on a stack of unfinished changes that you can reapply at any time.
拉取远程仓库
第二步，继续 pull 拉取远程仓库并进行自动合并：
git pull
重新还原暂存
第三步，stash pop 重新还原暂存修改：
git stash pop
最后一步，如果还原后产生冲突，需要手工或使用 mergetool 进行处理。处理完毕后，使用 add 标明冲突已解决：
git add .
