基于Cordova框架的Hyrid应用初解一

发表于 2016-04-07   |   分类于 Objective-C   |     |   次阅读

概述

通过Cordova框架来做H5和Native的混合应用。各方案优劣相信都有比较过。首先最主要的优势就是从App Store的审核周期中脱离出来了，我们可以打特定的补丁而不用加急审核了，在应用中可以实时展示页面，甚至访问本地数据、设备传感器等，同时也做到了一处编译跨平台运行的梦想。安装包精简这种附带的好处都是附赠的了。看完前面后面也得瞅瞅。既然是H5的网络就是一个障碍，与Native的界面相比流畅度有一定不足。

config.xml

Apache 科爾多瓦的應用程式都依賴于一個共同的 config.xml 檔，提供有關應用程式的資訊，並指定參數影響它如何工作，如它是否回應方向轉移。 此檔遵循 W3C 的打包 Web 應用程式或構件，規範。

應用程式本身作為 web 頁上，預設情況下名為index.html，引用任何 CSS、 JavaScript、 圖像、 媒體檔案或其他資源的本地檔是必要的它的運行來實現。 這款應用程式作為web 視圖的本機應用程式中包裝，您分發給應用程式商店內執行。

在config.xml文件中我们要进行一定的本地配置:

@interface MainViewController : CDVViewController
@end
@implementation MainViewController

  • (void)viewDidLoad {
    self.wwwFolderName = @”www/Pooh”;
    self.startPage = @”index.html#index”;
    [super viewDidLoad];
    }
    @end

  • (void)viewDidLoad {
    self.wwwFolderName = [NSString stringWithFormat:@”file://%@/www/Pooh”,[NSSearchPathForDirectoriesInDomains(NSDocumentDirectory, NSUserDomainMask, YES) objectAtIndex:0]];
    //self.wwwFolderName = @”www/Pooh”;
    self.startPage = @”index.html#index”;
    [super viewDidLoad];
    }
    ```
