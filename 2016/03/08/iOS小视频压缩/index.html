iOS小视频压缩

发表于 2016-03-08   |   分类于 Objective-C   |     |   次阅读

所见所分享。记录眼前的美好事物并通过我们的App进行展示。作为开发者，满足用户多长时间的拍摄时间为宜，15秒还是像微博取消了140字限制一样给个1分钟的拍摄时间。理论上可以长久录制视频，但克制才是真爱，如何为1分钟的视频压缩优化？

项目应用AVFoundation

捕获视频主要为AVCaptureSession：

  • objc-在iOS上捕获视频
  • 模仿快手应用的视频录制

基本的视频录制通过以上博文和Demo即可做出，其中：

_captureSession=[[AVCaptureSession alloc]init];
if ([_captureSession canSetSessionPreset:AVCaptureSessionPreset640x480]) {//设置分辨率
    _captureSession.sessionPreset=AVCaptureSessionPreset640x480;
}

AVCaptureSessionPreset可以设置不同的采样分辨率，此处基于清晰度选择640*480。为什么不选择High?

动辄上百兆的空间占用，还给你报个Memory received warning！ 选640*480清晰度达到要求，视频采集数据只有不到30M的占用，世界清静了。

视频输出

输入配置完了，现在把我们的注意力转向 capture session 的输出。

  • AVCaptureMovieFileOutput

压缩方案一

一般通过AVAssetExportSession进行视频压缩。当presetName选择为MediumQuality时基本1分钟的视频压缩下来为5M左右，我们在面向google编程的过程中发现网上大量的使用的

  • presetName:AVAssetExportPresetLowQuality

在presetName为LowQuality时视频是做到了极致的压缩，1分钟的视频不到1M的容量，但是压缩出来的视频充满了不规则的马赛克，给我们的观感造成极大的冲击，真爱生命远离打码。

基本的要求在这就满足，可以通过mergrFileURL来压缩上传我们拍摄的视频了。

不到的5M的上传容量，剩下的只有网速了。期待你来一句：我已准备好网速，你告诉我你是3G。还能不能愉快的玩耍了。现在5M不可取，那么只能重新编码了，做到2~3M的样子，保证视频清晰度的同时做到上传容量袖珍。

以上种种都是在AVCaptureDataOutput 和 AVAssetWriter帮助下完成对视频音频的自定义。如果你想要对影音输出有更多的操作，你可以使用 AVCaptureVideoDataOutput 和 AVCaptureAudioDataOutput 而不是我们方案一中展示的AVCaptureMovieFileOutput。
