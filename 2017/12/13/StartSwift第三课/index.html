关于此篇

前两篇介绍了Swift开发的基础姿势和列表页的创建，感兴趣童鞋可以点击这里查看。在iOS开发常见场景就是从一个页面跳转到另一个页面了，大部分这样的跳转还伴随着界面传值。本篇就着这一场景向大家介绍Swift语言中界面push、传值、网络请求及加载动画等使用姿势。

Push及传值

从一个Controller跳转到另一个Controller时，一般利用UINavigationController，调用pushViewController进行跳转，使用方式比较简便，实例化后交给navigationController即可。

我们发现SportDetailViewController有个detailModel的属性，从代码可推断是一个数据model，让我们来看看它的定义：

从类型定义上，SportDetailViewController会获取一个title的属性和url的链接进行处理，具体用它们来做什么呢，让我们开启push传送门

SportDetailViewController

天星降世，时有人于降落SportDetailViewController中发现两属性，一曰webView, 一曰SportDetailModel。研习之，发现controller拿detailModel中的title当了自己的门牌号，webView拿detailModel中的url当了自己的升云梯。

Loading

WebView是一个内嵌浏览器控件，加载网址URL时或多或少有等待时间，遇上网上特别慢时能急到便秘，有个加载动画抚慰下焦躁的心情是个不错的选择。前人栽树，后人纳凉，引入轻量加载动画框架如下

使用姿势如下代码所示： 


extension SportDetailViewController:UIWebViewDelegate {
    //MARK: webViewDelegate
    func webViewDidStartLoad(_ webView: UIWebView) {
        view.hud.showLoading("加载中")
    }

    func webViewDidFinishLoad(_ webView: UIWebView) {
        view.hud.dismiss()
    }

    func webView(_ webView: UIWebView, didFailLoadWithError error: Error) {
        view.hud.dismiss()
        //
    }  
}

至此，一套功守道是打完了，该喝咖啡了。
