关于此篇

上一篇介绍了一点Swift开发的基础姿势，感兴趣童鞋可以点击这里查看。在iOS中有代表性的基础页面就是列表页了，在OC中是tableView，Swift中也一样，这篇主要通过一个列表demo来看看Swift的基本开发姿势，与我们使用OC语言开发时具体有什么不一样呢，咱们往下走。

TableView的使用姿势

extension

扩展是给已经存在的类（class），结构体（structure），枚举类型（enumeration）和协议（protocol）增加新的功能。其中一个用法如下代码所示

extension ViewController: UITableViewDelegate, UITableViewDataSource {
//MARK: LifeCycle
override func viewDidLoad() {
super.viewDidLoad()
// Do any additional setup after loading the view, typically from a nib.
self.tableView?.rowHeight = 54.0
}

//MARK: TableViewMethods
func numberOfSections(in tableView: UITableView) -> Int {
    return 1
}

func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
    return items.count
}

func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
    let cell = tableView.dequeueReusableCell(withIdentifier: "Cell", for: indexPath) as UITableViewCell
    cell.accessoryType = .none
    cell.textLabel?.text = itemTitle[items[indexPath.row]]
    cell.imageView?.image = UIImage(named: "summericons_100px_0\(items[indexPath.row]).png")
    return cell
}

func tableView(_ tableView: UITableView, didSelectRowAt indexPath: IndexPath) {
    tableView.deselectRow(at: indexPath, animated: true)
    showItem(items[indexPath.row])
}

这里面我们发现有个\， 它有什么作用呢？

  • Swift在字符串中插入变量名作为占位符，使用反斜杠()来提示Swift替换变量/常量名为其实际的值。

套入上面的代码，我们可以取得的image的name分别为：summericons_100px_05.png, summericons_100px_06.png和summericons_100px_07.png
顺便给tableView设置个54的行高。至此，我们的列表页就完成的差不多了。可以把程序run起来看看效果了。
